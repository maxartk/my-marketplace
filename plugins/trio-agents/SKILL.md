# Trio Agents — Claude + Hermes + OpenClaw

Цей скіл описує як три агенти працюють разом як єдина команда.
Кожен агент має свою роль — разом вони закривають повний цикл задач.

## Ролі агентів

### 🧠 Claude (Anthropic)
- Глибокий аналіз, написання коду, документація
- Технічні пояснення, дебаг, code review
- Довгі складні задачі що потребують міркування
- Робота з файлами, репозиторіями, артефактами

### ⚡ OpenClaw
- Виконання автоматизацій та workflow
- Запуск скриптів, bash команди, cron задачі
- Управління файловою системою та конфігами
- Інтеграція з n8n, API, зовнішніми сервісами

### 🪁 Hermes
- Швидкі відповіді та довідка
- Маршрутизація задач між агентами
- Моніторинг та сповіщення
- Легкі задачі що не потребують важких моделей

## Тригери (коли завантажувати цей скіл)
Якщо користувач каже "всі агенти", "claude і openclaw", "trio", "команда агентів",
"передай claude", "хай hermes зробить" — завантаж цей скіл.

## Виклик агентів

```
@claude   — аналіз, код, документація
@openclaw — виконання, автоматизація, сервер
@hermes   — швидкі задачі, моніторинг, маршрутизація
```

## Залежності
Цей скіл використовує:
- `claude` — для аналізу та написання коду
- `openclaw` — для виконання команд та автоматизацій

## Правила маршрутизації

| Задача | Агент |
|--------|-------|
| Написати / проаналізувати код | Claude |
| Виконати команду на сервері | OpenClaw |
| Просте питання / довідка | Hermes |
| Надіслати повідомлення | OpenClaw |
| Дебаг складної помилки | Claude |
| Запустити автоматизацію | OpenClaw |
| Скласти документацію | Claude |
| Моніторинг / перевірка статусу | Hermes |

## Передача контексту між агентами

Коли один агент передає задачу іншому — включай:
1. Що вже зроблено
2. Що потрібно зробити далі
3. Важливий контекст (файли, змінні, результати)

Приклад:
```
[Claude → OpenClaw]
Код написаний і збережений в /tmp/fix.py
Потрібно: виконати на OCI сервері та перезапустити сервіс
Контекст: сервіс називається signal_bridge_remote
```

## Виклик іншого скілу
Якщо потрібен пошук інформації під час роботи:
1. Завантаж скіл `wikipedia-search`
2. Виконай пошук через його API
3. Передай результат потрібному агенту

## Сценарії спільної роботи

### 📱 Розробка Flutter додатку
```
1. @claude   — написати код / виправити баг
2. @openclaw — запустити flutter build apk
3. @openclaw — завантажити APK через GitHub Actions
4. @hermes   — повідомити що APK готовий
```

### 🔧 Автоматизація та DevOps
```
1. @hermes   — отримати задачу від користувача
2. @claude   — написати скрипт або конфіг
3. @openclaw — виконати скрипт на сервері
4. @hermes   — доповісти результат
```

### 📊 Дослідження та звіти
```
1. @hermes   — прийняти запит
2. @claude   — знайти інформацію, проаналізувати
3. @claude   — написати структурований звіт
4. @openclaw — зберегти в Google Drive або відправити
```

### 🐛 Дебаг та виправлення
```
1. @openclaw — зібрати логи з сервера/пристрою
2. @claude   — проаналізувати логи, знайти причину
3. @claude   — написати фікс
4. @openclaw — застосувати фікс, перевірити
5. @hermes   — повідомити про результат
```
